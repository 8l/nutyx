Post-construction and configuration of NuTyX

All the packages of a base system are now available. We can finalise and boot on NuTyX

  1. We start to re-enter in the NuTyX
    2. exit
  2. We re-define if needed the LFS variable
    3. echo $LFS
    must return: 
    /mnt/lfs
  3. We can now enter in the new NuTyX to finalise the job
    4. chroot $LFS /usr/bin/env -i HOME=/root TERM="$TERM" PS1='\u: \w\$' \
/bin/bash --login
    No error should be print: 
    root [ / ]#
  4. We can delete now the link bin to take advantage of the final scripts
    5. cd
rm bin
  5. We produce an new initrd needed for the boot and the final configuration
    6. mkinitramfs
  6. We add an entry to the menu.lst or grub.cfg file regarding the version of the grub
title NuTyX on sda2
    root (hd0,1)
    kernel /boot/kernel root=/dev/sda2 ro quiet vga=791
    initrd /boot/initrd
If you have allready grub 2 installed, just modify your grub.cfg like: 
menuentry "NuTyX on /dev/sda2" {
    set root=(hd0,2)
    linux /boot/kernel root=/dev/sda2 ro quiet
    initrd /boot/initrd
    }

    menuentry "Windows 8" {
    set root=(hd0,3)
    chainloader (hd0,3)+1
    }
If your NuTyX boot correctly and the network works, you can safely delete the /sources and /tools folders 
 rm -r /{sources,tools}


Congratulations, you have now a new base system NuTyX in your hands ready to use.
