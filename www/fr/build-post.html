    Post-construction et génération de l'iso

    Tous les paquets d'un système de base sont désormais disponible. On peut finaliser et produire une ISO installable de notre NuTyX

    1. On commence par sortir de la NuTyX
      2. exit

      On revérifie la variable LFS

      echo $LFS

      Elle doit retourner quelque chose:

      /mnt/lfs
    2. On peut maintenant réentrer dans la nouvelle NuTyX pour finaliser les travaux
      3. chroot $LFS /usr/bin/env -i HOME=/root TERM="$TERM" PS1='\u: \w\$' \
/bin/bash --login
      Aucune erreur ne doit apparaître: 
      root [ / ]#
    3. On commence par adapter le fichier cards.conf. En effet, on utilisera désormais uniquement les recettes issues du projet git. Adaptez si necessaire la langue et l'arch (i686 si en 32 bits).
      4. cat > /etc/cards.conf << "EOF"
dir /root/saravane/base
dir /root/saravane/system
logdir /var/log/pkgbuild
arch x86_64
locale fr
base /root/saravane/system
EOF
    4. On met à jour les fichiers MD5SUM des recettes se trouvant dans le dossier /root/saravane/base
      5. cd /root/saravane/base
for i in *
do
cd $i
pkgmk -us||break
cd ..
done
      ...
..

    6. On ajoute les fichiers necessaires au fonctionnement de l'iso
      7. cp -av /root/saravane/iso /ISO

      cp -v /boot/initrd /ISO/isolinux/initrd
      '/boot/initrd' -> '/ISO/isolinux/initrd'
    7. On produit maintenant les fichiers squashfs qui seront utilisés par l'iso
      8. mkdir -p /ISO/isolinux/boot/
cd /
for dir in opt bin etc lib root run sbin usr var
do 
  mksquashfs $dir ISO/isolinux/boot/$dir.squashfs
done
    8. On peut maintenant sortir de la saravane pour produire l'iso.
      9. exit
    9. On peut enfin générer notre ISO dont la version est à la date du jour.
      10. Avant de lancer la prochaine commande, assurez-vous que vous avez bien la variable LFS configurée et que les commandes mkisofs et isohybrid sont à disposition sur votre linux principale. Elles sont respectivement dans les paquets cdrkit et syslinux.
      sh $LFS/root/saravane/scripts/mkiso saravane-`date +%Y%m%d`

    Félicitations, vous avez maintenant entre les mains une ISO NuTyX de base prête à être installée sur toutes vos machines.

    Options

    1. Si vous souhaitez produire une iso contenant vos applications favorites, vous pouvez les compiler et les installer avant de produire l'iso. Dans l'exemple ci-dessus, c'est l'interface graphique xorg qui sera ajoutée
      2. cd /root/saravane/base
cards depends xorg > .list
echo "xorg" >> .list
pass

      La compilation commence:

      =======> PKGMK_INSTALL: install
=======> PKGMK_WORK_DIR: /root/saravane/base/elfutils/work
=======> PKGMK_SOURCE_DIR: /root/saravane/base/elfutils
=======> PKGMK_KEEP_SOURCES: no
=======> PKGMK_CLEAN: yes
=======> WARNING: FOOTPRINT AND MD5SUM IGNORED
=======> MD5SUM file will be generated
=======> PKGMK_COMPRESS_PACKAGE: no
=======> name: elfutils
=======> version: 0.161

      Une fois la compilation terminée, vous pouvez soit choisir l'option ci-dessous, soit continuer la génération des fichiers squashfs au point 7.

    2. Si vous souhaitez réduire la taille finale de l'ISO, vous pouvez supprimer les binaires et les logs produits. Si vous souhaitez les conservez, lancez la commande ci-dessus.
      3. rsync -a /root/saravane/system /system

      Aucune erreur ne doit apparaître

      On peut maintenant effacer les binaires du git

      find /root/saravane/system/ -name "*.cards.*" -exec rm {} \;

      Aucune erreur ne doit apparaître

      On efface les logs.

      mv /root/log /log

      Si besoin est, les binaires et les logs se trouvent donc à la racine de la NuTyX
